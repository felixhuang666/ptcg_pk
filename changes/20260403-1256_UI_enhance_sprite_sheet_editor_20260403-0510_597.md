# Enhance Sprite Sheet Editor: Add Output Row and Column Controls

**Task Information:**
- TID: 597
- Original Branch Commit ID: d9eb4b772d5182051d742b548d4c9663e451c330
- Date/Time: 20260403-0510

**User Prompt:**
Objective:enhance sprite sheet editor
- in output menu tab, add output rows and columns control to tune the output png and json.

**Root Cause (Current State):**
The sprite sheet generation logic used hardcoded values for dimensions: a maximum of 10 columns and dynamically calculated rows. It didn't allow users to tune the output PNG and JSON grid dimensions.

**Solution:**
1. Added `outputCols` (default: 10) and `outputRows` (default: 1) state variables in `SpriteSheetEditor.tsx`.
2. Added a `useEffect` hook to automatically expand the `outputRows` when the user adds more items to the queue than the current grid size capacity.
3. Added "Cols" and "Rows" input components to the "OUTPUT" tab UI.
4. Updated the `generateSpriteSheet` calculation to strictly respect the `outputCols` and `outputRows` state dimensions. Processing handles `Math.min(outputQueue.length, outputCols * outputRows)` to prevent index errors.
5. Updated the "JSON" preview tab to reflect the new user-defined layout dimensions.
6. Updated the `loadExistingTileset` function to calculate and set the exact `outputCols` and `outputRows` state when loading a preexisting tileset.
