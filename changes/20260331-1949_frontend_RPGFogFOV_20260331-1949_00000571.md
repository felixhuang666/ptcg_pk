## Root Cause
The RPG mode previously rendered the entire map based on tilemap data without a constrained field-of-view (FOV) logic, nor did it filter out updates from distant players. To satisfy performance and design requirements, a fog of war constrained to a specific distance is required.

## Solution
1. Added `VITE_RPG_CAMERA_SIZE=10` to `.env.example` as requested to configure the default camera/FOV size.
2. In `src/components/RpgMode.tsx`, implemented a Fog of War overlay (`fogGraphics`) using `Phaser.GameObjects.Graphics`. It draws large black rectangles spanning outwards from the `VITE_RPG_CAMERA_SIZE` bound area.
3. Updated the `update` loop logic to iterate over all active `otherPlayers` and `npcs`, calculating their distance to the local `player`. Any entity located further than the `VITE_RPG_CAMERA_SIZE` distance is hidden (`setVisible(false)`).
4. Modified the `chat_message` socket handler to explicitly ignore rendering new chat bubbles if the emitting player is located outside the active FOV boundary.
