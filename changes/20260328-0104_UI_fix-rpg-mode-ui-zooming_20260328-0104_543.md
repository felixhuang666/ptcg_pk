**Root Cause:**
In `src/components/RpgMode.tsx`, the map coordinates (`infoText`), mode button (`modeButton`), and attack button (`attackButton`) were rendered as Phaser Text objects. While they used `setScrollFactor(0)` to stay fixed relative to the camera's scroll position, they were still affected by the camera's zoom level (`this.cameras.main.zoom`). Thus, when the user zoomed in, the UI elements scaled up and moved partially off-screen. Additionally, the map coordinates were positioned at the top-right instead of the requested top-left.

**Solution:**
1. Removed `infoText`, `modeButton`, and `attackButton` objects from the Phaser `MainScene`.
2. Created React DOM overlays using absolute positioning (`absolute inset-0`) mapped over the Phaser game container.
3. Added a `div` at the `top-left` (`top-2 left-2`) to display the coordinates, updated via `infoTextRef.current.innerText` directly in the Phaser `update()` loop to ensure optimal performance (avoiding 60fps React re-renders).
4. Added the action buttons at the `bottom-left` (`bottom-4 left-4`), managing the action mode via React state and communicating click interactions to the Phaser scene via a mutable ref (`mainSceneRef`).
5. This completely decouples the UI elements from the game camera's zoom functionality.
