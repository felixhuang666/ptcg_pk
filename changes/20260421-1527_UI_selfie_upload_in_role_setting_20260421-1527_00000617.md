# Selfie Upload in Role Setting

**TID:** 617
**CHANGE_PREFIX:** 20260421-1527

## User Prompt

可上傳玩家照片到角色設定
in 角色設定, player can rename his alias.
I would like to let palyer able to turn camera in web (ios safari) and take a selfie.
it must have a preivew windows and a button to confirm the selfie.
save player's selfie to local asset (public/assets/players_pic/{uid}.png)
when player enter the 角色設定 page, must load it from public/assets/players_pic/{uid}.png

## Root Cause
The Role Setting page lacked the capability to capture a photo using the device camera and the backend did not support receiving and saving the image as a local asset associated with the user ID.

## Solution
1. Frontend updates to `RoleSetting.tsx`:
   - Utilized `navigator.mediaDevices.getUserMedia({ video: { facingMode: 'user' } })` to request camera access.
   - Fixed camera preview attachment by utilizing a `useEffect` hook tracking `showCamera` and `cameraStream` state.
   - Rendered the camera stream in a `<video autoPlay playsInline>` element for live preview.
   - Used an offscreen canvas to capture a snapshot of the video. The canvas context is scaled horizontally by -1 so the captured image is not mirrored, matching the user's perception of the mirrored preview.
   - Combined the snapshot capturing and uploading sequence into a single "拍照並上傳" button based on user feedback.
   - Added a "照片" section to display the current profile picture from `/assets/players_pic/{uid}.png`, and a fallback image if it fails to load. A timestamp cache buster is used to update the image immediately after upload.
2. Backend updates to `backend/main.py`:
   - Created a `POST /api/user/selfie` endpoint.
   - Authenticated user by `session_id`, fetching the `user_data` to get the `oauth_data.id` (which is the uid).
   - Read the base64 encoded image data from request body and checked for basic size constraints.
   - Saved the decoded PNG data into `public/assets/players_pic/{uid}.png` and `dist/assets/players_pic/{uid}.png` so that the file is immediately available in both environments.
