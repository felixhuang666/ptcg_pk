## Root Cause
The backend test cases (`backend/tests/test_api_maps.py`, `backend/tests/test_api_maps_resize.py`, and `backend/tests/test_real_api.py`) created mock maps and dynamically generated maps in Supabase but lacked teardown logic to delete them after execution. This led to a buildup of test data in the `maps` table and `in_memory_maps`.

## Solution
1. Implemented a new `DELETE /api/map/{map_id}` endpoint in `backend/main.py` to support deleting a map both from the `in_memory_maps` global dictionary and the Supabase `maps` table.
2. Updated the test files to invoke this new endpoint at the end of relevant tests:
   - `test_api_maps.py`: Cleans up the test map `test_map_001` and the generated map beginning with `gen_`.
   - `test_api_maps_resize.py`: Cleans up the test map `test_map_resized`.
   - `test_real_api.py`: Cleans up the dynamically generated map created during the test.
3. Successfully executed `make test` to ensure all tests passed and maps were properly cleaned up.
