## Root Cause
The `RpgSceneEditor` historically only supported a single flat array (`map_list`) in the backend/frontend logic. There was no concept of organizing maps into logical sub-groups (layers), and no way to specify the layer identity for individual maps or manage these layers in the UI.

## Solution
1. **Model & State Updates**:
   - Introduced a `layers` array to `sceneData` storing layer metadata (e.g., `{ id, name }`).
   - Added `activeLayerId` state to the React component to track the current layer selection.
   - Initialized a default `Base Layer` for new scenes and automatically backfilled existing loaded scenes if layers were missing.

2. **Layers UI Panel**:
   - Added a "Layers" list in the left sidebar to add, select, and delete layers. Selecting a layer changes the current working target.

3. **Drag & Drop and Inspector Tools**:
   - Maps dragged into the canvas automatically receive the `layer_id` of the `activeLayerId`.
   - Updated the properties inspector (Right Sidebar) to allow viewing and switching the `layer_id` for individual maps dynamically.

4. **Phaser Rendering Integration**:
   - Refactored the Phaser scene's rendering process so that maps are sorted based on their mapped layer's position, ensuring they are rendered in the correct sequential order.
