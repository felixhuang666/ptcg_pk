## Root Cause / Context
The application previously had no authentication system, meaning players couldn't save their progress, scores, or identities persistently across sessions or devices. The user requested Google OAuth integration to solve this.

## Solution
1. **Supabase Integration**: Installed `@supabase/supabase-js`, `@supabase/auth-ui-react`, and `@supabase/auth-ui-shared` using `--legacy-peer-deps` to respect the existing React 19 setup.
2. **Auth Component**: Created `src/components/Auth.tsx` to display the Supabase login UI for Google OAuth, complete with graceful fallback if environment variables are not yet configured.
3. **App State**: Updated the Zustand store (`src/store/appStore.ts`) to track the authenticated `user` state.
4. **App Integration**: Wrapped the main application logic in `App.tsx` with an auth check, conditionally rendering the `<Auth />` component for unauthenticated users and listening to Supabase auth events. Also added a logout mechanism.
5. **Environment Configuration**: Added placeholders `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY` to `.env.example`. Added TS typings for `import.meta.env` to `tsconfig.json`.
6. **Documentation**: Created a detailed setup guide at `docs/setup_oauth.md` to instruct users on how to configure Google Cloud and Supabase to enable OAuth.
