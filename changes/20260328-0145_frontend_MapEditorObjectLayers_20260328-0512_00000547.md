# Map Editor Multiple Layers & Object Support

- **Prompt**: "map generation is incorrect, i expected that it should generate and fill object to the map. when load the map, i cannot see the objects (such as grass picture)"
- **Original Branch Commit ID**: 000380ceca3970f180affbe412c0335a12a3c787
- **Root Cause**: Procedural map generation only supplied a single array (`tiles`) rendering solid tiles (like dirt, water, mountain) and overwriting each other, which prevented overlapping transparent objects (like trees and bushes on top of grass). The Phaser Editor UI was unaware of an object layer.
- **Solution**:
  1. Updated the FastAPI `/api/map/generate` endpoint to calculate an independent `objects` array alongside `tiles`, intelligently populating rocks on dirt, bushes on grass, and mapping everything via the `cute_tileset.json` IDs. Empty spaces return `-1`.
  2. Integrated two Phaser layers using `this.tilemap.createBlankLayer('ground')` and `('objects')` instead of passing data flat. Adjusted collision detection cleanly separated between water and object boundaries.
  3. Upgraded the React Editor UI to feature a Layer Selection `<select>` (Ground vs Object) and an Eraser tool `<button>` utilizing the backend arrays respectively. Exposing these correctly synced variables fixes painting functionality while allowing users to place objects or remove them over existing ground safely.
