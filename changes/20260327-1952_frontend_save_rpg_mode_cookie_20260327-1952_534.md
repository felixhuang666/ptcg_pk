**Root Cause:**
The frontend application previously reset to the `MENU` view upon every reload. It lacked a mechanism to remember the last active view, specifically if the user was playing in `RPG_MODE`.

**Solution:**
1. Modified `src/App.tsx`'s initial state for `view` to lazily evaluate `document.cookie`. If a `last_view` cookie is found with the value `RPG_MODE`, the app will automatically render the RPG mode directly.
2. Implemented a `useEffect` hook in `src/App.tsx` that monitors changes to the `view` state. Whenever the view changes to `RPG_MODE`, a `last_view=RPG_MODE` cookie is saved with a one-year expiry. If the view changes to anything else, the cookie is deleted.
3. Added frontend verification using Playwright to confirm the cookie behavior is working as intended.
4. Updated `docs/ui_design.md` to reflect the UI state persistence across sessions.
