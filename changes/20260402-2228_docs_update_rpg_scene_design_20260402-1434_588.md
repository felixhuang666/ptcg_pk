## Root Cause / Context
The original `docs/rpg_scene_design.md` was a brief work-in-progress outline. It needed detailed architectural designs for combining maps into scenes, layer integration, NPC handling, events, and a unified editor UI.

## Solution / Changes Implemented
* **Updated `docs/rpg_scene_design.md`:**
  * **RPG Scene Structure:** Detailed the `game_scene` table schema to include `scene_entities` along with the `map_list` arrays, establishing a macro-level coordinate space.
  * **Combining Editors:** Added a section on combining the Map Editor and Scene Editor through a "Scope Toggle" and global grid, allowing both high-level scene composition and in-place local map editing.
  * **Layer Integration:** Described how the 6 static map layers integrate with a new virtual 7th "Dynamic Scene Layer" for entities, utilizing Y-coordinate based depth sorting.
  * **Game Objects & Interactions:** Outlined interaction models (On Collision, On Action) and prefab dragging mechanisms.
  * **NPC Editor:** Detailed NPC placement, patrol pathfinding (waypoint tools), and dialog tree editors.
  * **UI Design:** Enhanced the UI specifications to include Outliners, Palettes, and context-sensitive Property Inspectors handling various entity types.
