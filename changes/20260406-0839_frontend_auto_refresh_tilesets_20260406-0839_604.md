# 20260406-0839 - Frontend - Auto refresh tilesets in RpgMapEditor

## Input Prompt
Objective:auto refresh /reload the tileset when enter the map editor
- when user enter the map editor, it will load the first map. but it doesn't reload the available tilesets.

## Original Branch Commit ID
`d34d0b3f2e7b07883bbb61c8cb16de697e663ee3`

## Root Cause
The `loadTilesets` method inside `RpgMapEditor.tsx` fetches available tilesets using the `/api/map/tilesets` endpoint.
1. The browser caches the `GET` request, meaning if the user modifies tilesets elsewhere (like in the Sprite Sheet Editor) and then returns to the map editor, they get stale data.
2. The method was only called on the initial mount (`useEffect([], ...)`). While navigating views normally triggers a re-mount, there was no explicit hook triggering `loadTilesets()` when `handleMapLoaded` triggers an inner map state load.

## Solution
1. **Cache Busting**: Added `?t=${Date.now()}` to the fetch URL inside `loadTilesets` (`/api/map/tilesets?t=${Date.now()}`) to force the browser to always fetch fresh data.
2. **Reload on Map Load**: Updated the `handleMapLoaded` listener to additionally invoke `loadTilesets()`, ensuring that tileset selection is strictly reloaded and updated every time a new map layout enters the active viewport.
