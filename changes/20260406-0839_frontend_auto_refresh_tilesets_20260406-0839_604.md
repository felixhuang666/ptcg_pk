## Root Cause
The `loadTilesets` method inside `RpgMapEditor.tsx` fetches available tilesets using the `/api/map/tilesets` endpoint.
1. The browser caches the `GET` request, meaning if the user modifies tilesets elsewhere (like in the Sprite Sheet Editor) and then returns to the map editor, they get stale data.
2. The method was only called on the initial mount (`useEffect([], ...)`). While navigating views normally triggers a re-mount, there was no explicit hook triggering `loadTilesets()` when `handleMapLoaded` triggers an inner map state load.
3. Additionally, inside the Phaser scene's `create()` method, the initial map fetch populated the map data, but failed to dispatch the `mapLoaded` event to the React container. Consequently, the React UI never received the trigger to re-render the left sidebar matching the newly loaded map tilesets.
4. Finally, when the tilesets populated the left sidebar and they were larger than the viewport, the wrapper missing `min-h-0` and dynamic shrink configurations prevented the CSS vertical overflow `overflow-y-auto` from functioning.

## Solution
1. **Cache Busting**: Added `?t=${Date.now()}` to the fetch URL inside `loadTilesets` (`/api/map/tilesets?t=${Date.now()}`) to force the browser to always fetch fresh data.
2. **Reload on Map Load**: Updated the `handleMapLoaded` listener to additionally invoke `loadTilesets()`, ensuring that tileset selection is strictly reloaded and updated every time a new map layout enters the active viewport.
3. **Dispatch Initialization Event**: Added `window.dispatchEvent(new CustomEvent('mapLoaded', ...))` after the initial fetch in the Phaser `create()` method. This properly cascades the event to React, triggering the sidebar re-render immediately upon entering the Map Editor view.
4. **Scrollbar Support**: Set `min-h-0` on the tiles wrapper class to allow vertical scrolling within the flex layout (`<div className="flex-1 overflow-y-auto p-2 min-h-0">`), and ensured the map tileset tabs above it wrap correctly using `overflow-x-auto` to prevent horizontally expanding parent nodes.
