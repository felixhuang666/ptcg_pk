# RpgMode Scene Map Merging Fix

**TID:** 624
**Change Prefix:** 20260424-1158

## Summary
Fixed an issue in RPG mode where switching to a scene (e.g., `local_scene_1`) containing multiple map instances failed to render the maps properly.

## User Prompt
在RPG模式下, 切換quest中的scene仍然沒法正常顯示地圖
i try to switch to local_scene_1 (which is from local asset), the map cannot be rendered correctly.

## Root Cause
In `RpgMode.tsx`, the logic to parse a scene's map list was incomplete. It was missing maps that were nested inside `sceneData.scene_entities.map_list` (as documented in technical debt) and only ever rendered the *first* map of the scene's list, completely ignoring the layout (`offset_position`), size, and rendering of all subsequent maps within the same scene.

## Solution
1. Added `loadAndMergeSceneMaps` helper to `RpgMode.tsx` to handle parsing the map list from either top-level `sceneData` or nested `scene_entities`.
2. Modified the helper to concurrently fetch all map configurations listed in the scene.
3. Implemented merging logic that calculates the bounding box of the entire scene across all positioned maps and composites their tile data and tilesets into a single virtual `mapData` object.
4. Refactored the `create` and `loadNewMap` methods in `RpgMode.tsx` to call this new helper when fetching a scene, thereby enabling correct rendering of multi-map scenes in RPG mode.
