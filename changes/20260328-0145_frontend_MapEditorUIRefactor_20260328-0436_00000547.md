# Map Editor UI Refactor & API Tests

- **Prompt**: "reactor UI: move save map and other control key/button to tool bar. when click save map, should sync map to supabase. add pytest test cases to check backend API, also create github CI yaml to test backend API after a PR push"
- **Original Branch Commit ID**: 000380ceca3970f180affbe412c0335a12a3c787
- **Root Cause**: The Map Editor UI inside RpgMode was partially rendered inside the Phaser canvas (`setupEditorUI()`), which did not match the newly requested responsive React Top Toolbar paradigm, restricting feature scale and styling. Furthermore, robust validation and automated continuous integration for the backend API logic was missing.
- **Solution**:
  1. Removed `setupEditorUI()`'s Phaser Canvas UI generation and integrated "Undo", "Redo", "Save Map", and the "Paint Selection" components directly into the React Toolbar element wrapping the `PhaserGame`.
  2. Linked the React `handleSaveMap` button to successfully sync to the backend API via POST (`/api/map`), which correctly inserts or updates map states into the Supabase database.
  3. Mapped Phaser keyboard shortcuts (Keys 1, 2, 3) to dispatch custom window events to React components to keep the selection UI indicator in sync.
  4. Authored `pytest-asyncio` cases inside `backend/tests/test_api_maps.py` completely covering Map endpoints `/api/maps`, `/api/map` retrieval, updating, and `/api/map/generate` procedural algorithms.
  5. Implemented a Github CI Pipeline (`.github/workflows/backend.yml`) configured to install dependencies via Python 3.12 and trigger Pytest upon pushing to main.
