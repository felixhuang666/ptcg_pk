# Map Editor Map Resizing Module

- **Prompt**: "新增可以設定地圖大小的功能: 可以設定寬幾格 高幾格. 設定後 按下存儲 會直接修改地圖資料 同步到supabase"
- **Original Branch Commit ID**: 000380ceca3970f180affbe412c0335a12a3c787
- **Root Cause**: The Map Editor was lacking a dynamic dimension adjustment tool. The previously generated boundaries (like 200x200) were rigidly hardcoded during the initial generation request and could not be expanded or cropped within the active project instance without wiping via regeneration.
- **Solution**:
  1. Updated the React Map Editor ToolBar by introducing `ResizeWidth` and `ResizeHeight` input variables corresponding to the loaded map size dynamically via the new `mapLoaded` CustomEvent.
  2. Built `resizeMapData()` internally in the `MainScene` Phaser structure, effectively reconstructing the existing flat 1-dimensional array arrays (`tiles` & `objects`) over to the exact new boundaries specified, appending grass tiles or generic empties (`-1`) to filled limits when increasing array limits, and effectively cropping array lengths when modifying indices inwards.
  3. Integrated the frontend logic inside the `handleResizeMap` function with the `handleSaveMap` command execution directly, allowing dimension adjustments and saving events to happen instantaneously via POSTing the newly bound `mapData` objects to Supabase.
  4. Expanded backend `pytest` testing in `test_api_maps_resize.py` evaluating correct database inserts on differing property lengths.
