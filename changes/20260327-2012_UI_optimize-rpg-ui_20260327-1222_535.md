## Root Cause
- The mode switch buttons (Play / Edit) were awkwardly placed in a fixed floating bar at the bottom right.
- The chat box minimize/maximize functionality was implemented as a clickable title header in the chat box itself which lacked discoverability and consistent UI framing compared to the other top toolbar actions.

## Solution
- Imported `MessageSquare` icon from `lucide-react` to be used for the chat toggle UI element.
- Moved the `Play` and `Edit` mode switch buttons to the top header navigation toolbar, styling them inside a neat toggle group.
- Placed a new icon button in the header toolbar using `MessageSquare` to control the `isChatMinimized` state.
- Removed the old mode switch floating container from the bottom right of the screen.
- Removed the redundant click handler from the chat box header and cleaned up the conditional rendering of the chat window content to always display when not minimized.
