## Root Cause
The `rpg_mode` directory contained a prototype of a new RPG mode, but it was not integrated into the main React application (`src/`) or the main FastAPI backend (`backend/`). The goal was to migrate its functionality over.

## Solution
1. Moved assets (`character.png`, `character_atk.png`) from `rpg_mode/public` to `public/`.
2. Created `src/components/RpgMode.tsx` combining `rpg_mode/src/App.tsx` and `PhaserGame.tsx`.
3. Updated `src/App.tsx` to include an `RPG_MODE` view and a button to access it.
4. Installed the `phaser` dependency in `package.json`.
5. Added `GET /api/map` and `POST /api/map` endpoints in `backend/main.py` using `supabase.create_async_client` to persist the map, with an in-memory fallback.
6. Updated `backend/socket_app.py` to handle RPG mode specific multiplayer events (`rpg_connect`, `player_moved`, `player_left`).
7. Deleted the old `rpg_mode/` directory.
8. Added a migration script (`scripts/migrate_20260326-2114_create_maps_table.py`) to instruct users on creating the necessary `maps` table in Supabase.
9. Added documentation for the RPG mode in `docs/rpg_mode.md`.
