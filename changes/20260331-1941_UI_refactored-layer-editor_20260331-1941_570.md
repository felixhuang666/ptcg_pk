**Root Cause:**
Feature request to expand map editing capabilities from a simple 2-layer structure to a multi-layered functional map editor suitable for RPG maps, with visual boundaries, explicit collision layers, event trigger layers, and overhead elements.

**Solution:**
1. Refactored `RpgMapEditor.tsx` state to support 6 separate map layers: `base`, `decorations`, `obstacles`, `objectCollides`, `objectEvent`, `topLayer`.
2. Updated the left sidebar UI to conditionally render Tileset selection for visual layers (Base, Decorations, TopLayer) and simple numeric button selections for logic layers (Obstacles 0/1, ObjectCollides 0/10001-10010, ObjectEvent 0/20001-20010).
3. Migrated Map Data storage format from `{ tiles: number[], objects: number[] }` to `{ layers: Record<string, number[]> }`. Added an automatic map data upgrader (`upgradeMapData`) to maintain backwards compatibility with existing backend JSON maps.
4. Refactored Phaser Scene map instantiation (`renderMap`) to load 6 `TilemapLayer` instances with respective depths (0 to 10). Set collisions on `obstaclesLayer` and `objectCollidesLayer`.
5. Added a dynamic opacity check in the editor scene's `update` loop: when a camera (or pseudo-player area) is underneath a non-zero tile in the `topLayer`, `topLayer.setAlpha(0.5)` is triggered, making it transparent.
6. Refactored UI tools: Replaced `select` dropdown with distinct toggle icon buttons to switch between current edit layer modes.
