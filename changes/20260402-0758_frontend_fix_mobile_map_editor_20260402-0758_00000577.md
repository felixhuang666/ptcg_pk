## Prompt
> 用手機瀏覽器, map editor 無法正常看到地圖內容

## Root Cause
The `RpgMapEditor` component had sidebars (`showLeftSidebar` and `showRightSidebar`) enabled by default (`true`). On mobile devices with narrow screens, these sidebars occupied the entire viewport width, pushing the map editor canvas out of view or hiding it completely because they were displayed in a row without appropriate responsive positioning.

## Solution
1. Modified the initial state of the sidebars in `src/components/RpgMapEditor.tsx` to conditionally default to visible only on screens wider than 768px (`window.innerWidth > 768`).
2. Added Tailwind CSS classes to the sidebars (`absolute md:relative z-[6000] h-full max-h-full`) so that when they are opened on mobile devices, they overlay the map rather than participating in the flex layout and squashing the canvas.
3. Added the `relative` class to the parent `flex-row` container so the absolute-positioned sidebars are constrained properly.
