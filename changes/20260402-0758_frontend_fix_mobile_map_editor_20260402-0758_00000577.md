## Prompt
> 用手機瀏覽器, map editor 無法正常看到地圖內容
> regarding map editor, must support below operations in mobile browser:
> * move the map   (use IOS finger gesture)
> * zoom in map   (use IOS finger gesture)
> * zoom out map  (use IOS finger gesture)

## Root Cause
The `RpgMapEditor` component lacked the logic to support two-finger pinch-to-zoom multi-touch gestures and zoom-dependent panning. Additionally, its sidebars (`showLeftSidebar` and `showRightSidebar`) were enabled by default (`true`) in flex layout, causing them to occupy the entire viewport width on narrow screens and thus pushing the map editor canvas out of view.

## Solution
1. **Layout Fixing:** Modified the initial state of the sidebars in `src/components/RpgMapEditor.tsx` to conditionally default to visible only on screens wider than 768px.
2. **Overlay Menus:** Added Tailwind CSS absolute positioning classes to the sidebars (`absolute md:relative z-[6000]`) so that when they are toggled on mobile, they overlay the map rather than participating in the flex layout.
3. **Pinch-to-Zoom:** Implemented mobile pinch-to-zoom in the `MainScene` using Phaser multi-touch logic (`pointer1` and `pointer2`), calculating zoom ratio relative to the midpoint.
4. **Drag/Panning Speed Correction:** Scaled the map panning logic based on the current camera zoom level (`dx / this.cameras.main.zoom`) for consistent tracking.
5. **Mouse Wheel Zoom:** Included a complementary enhancement adding zooming with the scroll wheel for desktop clients.
