# 20260423-1713 Frontend Fix Phaser Collision Crash

- **TID**: 00000622
- **Summary**: Fix bug where switching scenes causes `Cannot read properties of undefined (reading 'tileWidth')` in Phaser.
- **Original branch commit id**: HEAD

## Root Cause
When switching scenes or re-rendering maps (`renderMap()`) in both `RpgMode.tsx` and `RpgMapEditor.tsx`, the existing tilemap and tilemap layers were being destroyed. However, the physics colliders linking the player sprite to those layers (created via `this.physics.add.collider(this.player, l)`) were not being destroyed. On the next frame, the Phaser physics engine attempted to evaluate these orphaned colliders, trying to access `l.tilemap.tileWidth` on the destroyed layer, resulting in the runtime crash.

## Solution
1. In `RpgMode.tsx` and `RpgMapEditor.tsx`, added a class array property `private layerColliders: Phaser.Physics.Arcade.Collider[] = [];` to track all colliders generated for map layers.
2. Modified the layer creation logic inside `setupLayers` to push the created collider objects to the array: `this.layerColliders.push(this.physics.add.collider(this.player, l));`.
3. Updated `renderMap()` to explicitly iterate over and destroy all tracked colliders (`this.layerColliders.forEach(c => c.destroy());`) and clear the array before destroying the tilemap and layers.
