## Prompt
Objective:refactor map editor

- adjust default map's size to 40x40;
- check current map's data (in-memory or in supabase), update the size (40x40) to all maps, include working maps
- add collapsible left side bar for map elements for admin to select elements to draw into to map
- add collapsible left side bar for detailed of selected the element, include the file name, positon (x,y,w,h) of the file name
- search files in map_tileset folder (dist/assets/map_tileset/xxxx_tileset.json) and render elements from the selected file (xxxx_tileset.png); xxxx_tileset.json defines elements data and pictures in xxxx_tileset.png
- add toggle icon button in tool bar to hide/show the left side bar and right side bar

## Root Cause
- The old map editor was rudimentary, lacking standard tileset palette tools, proper properties view, adjustable block sizing, and dynamically parsed tileset definitions.
- The map array bounds were originally hardcoded to 16x16 and 32x32 interchangeably instead of 40x40.
- Existing database maps needed reshaping to conform to the new 40x40 default.

## Solution
1. **Layout & UI Refactor:** Implemented a three-column layout in `RpgMapEditor.tsx`, consisting of a left `TilesetPanel`, a center `Phaser Canvas`, and a right `TileDetailsPanel`.
2. **Toggle Buttons:** Added intuitive toggle buttons (`<PanelLeftOpen>` and `<PanelRightOpen>`) to the top toolbar to collapse/expand the side panels.
3. **Dynamic Tilesets:** Created a FastAPI endpoint `/api/map/tilesets` to scan for and serve tileset JSON definitions from `dist/assets/map_tileset` (with fallback to `public/`).
4. **Tile Palettes:** Parsed the tileset structure to group sprites by tags/types. Implemented clickable tiles which calculate X, Y, W, H coordinates for the details panel.
5. **Map Size Standardized:** Standardized new map generation via `generate_map` in the backend to 40x40. Passed `resizeWidth` and `resizeHeight` parameters explicitly from the frontend toolbar.
6. **Block Sizing & Grid Toggle:** Added settings for modifying Block Size (W/H), defaulted to 32x32. Added a dynamic Phaser Grid overly (`scene.toggleGrid()`) to visually indicate block boundaries.
7. **Phaser Event Sync:** Established event syncing so selecting a tile in the React UI propagates to `MainScene.ts`, enabling proper brush selection and tile drawing logic.
8. **Database Migration:** Created a migration script `scripts/migrate_20260329_1422_resize_maps_40x40.py` to recursively iterate through all users' maps stored in Supabase JSONB, clipping or zero-padding the 1D layer arrays to 1600 elements (40 * 40).
