- **TID**: 560

## Prompt
"fix map render in RPG walk mode
- in RPG walk mode, the pic cannot be rendered correctly.
- In MAP editor, I can see elements in map are rendered, but I cannot see them in walk mode or battle mode."

## Root Cause
In `src/components/RpgMode.tsx` (the component responsible for RPG walk mode), the `renderMap()` function was hardcoded to only generate a basic 3-color ground layer, completely ignoring the `objects` array in the map data and the actual dynamically loaded tileset images (JSON/PNG). It lacked the complete layer rendering setup that was present in `src/components/RpgMapEditor.tsx`.

## Solution
1. Updated `renderMap()` and the nested `setupLayers()` functions in `RpgMode.tsx` to match the comprehensive logic used in `RpgMapEditor.tsx`.
2. Added logic to dynamically load tilesets (`loadAndRenderTileset`) based on the active map data.
3. Added the creation of the `object_layer` which processes and renders the `this.mapData.objects` array.
4. Ensured that the React component fetches the `/api/map/tilesets` metadata on mount, populating `window.__ACTIVE_TILESET__`, so that the Phaser game has the required textures before booting.
5. Visually verified through a Playwright E2E script and screenshots that the RPG Walk Mode now successfully displays the custom map tiles and objects, identical to the map editor.
