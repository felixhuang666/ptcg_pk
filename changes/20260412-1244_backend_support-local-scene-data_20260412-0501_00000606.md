# Feature: Support local scene data
- TID: 606
- Change Prefix: 20260412-1244
- Original branch commit id: Not applicable
- Root cause: Missing support to load and save scene, maps, and templates to local filesystem for better debugging.
- Solution:
  - Updated backend `get_scene`, `get_scenes`, `get_map`, `list_maps` to support falling back to local files (`dist/assets` and `public/assets`).
  - Added a backend `POST /api/save_local` endpoint to write these resources directly into the file system.
  - Added frontend `Save to Local Asset` feature in `RpgSceneEditor` to orchestrate this logic by bundling active scene and mapped data.
