# Generate UI design file in Figma format for RPG Map Editor

## Task Information
- TID: 578
- CHANGE_PREFIX: 20260402-0804

## Prompt
> visit current map editor code base and generate UI design file in Figma format

## Root Cause / Request
The user wants to generate a Figma design file from the current Map Editor UI. To do this, we need to launch the UI, navigate to the Map Editor component, and extract the DOM into a format that Figma can ingest.

## Solution
1. Installed `puppeteer` and `@builder.io/html-to-figma`.
2. Created a script that launches a headless browser, logs in as the dev tester, navigates to the Map Editor, and injects the `html-to-figma` library.
3. Extracted the DOM tree of the Map Editor into a `.json` format that is directly compatible with Figma (using the Builder.io Figma plugin or similar importers).
4. Saved the output to `docs/rpg_map_editor_figma.json`.

## Update: .pen Format Generation
After further requests from the user to study the `.pen` format documentation for Pencil (https://docs.pencil.dev/for-developers/the-pen-format), an additional step was added.

A transformation script (`figma_to_pen.cjs`) was created to parse the `docs/rpg_map_editor_figma.json` and convert it into the correct Pencil `.pen` schema. This involves mapping Figma JSON types (`FRAME`, `RECTANGLE`, `TEXT`, `VECTOR`) to `.pen` objects (`frame`, `rectangle`, `text`, `polygon`), handling IDs, background colors, alignments, and sizes, then storing it with `version: "2.9"`.

The new `.pen` file is saved alongside the previous JSON export as `docs/rpg_map_editor.pen`.
