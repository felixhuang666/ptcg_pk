# Handle favicon.ico correctly

**Task Information:**
- TID: 549
- CHANGE_PREFIX: 20260328-0921

**Prompt Summary:**
The user placed a `favicon.ico` in the root folder, but running `make build` didn't copy it to the `dist` folder. The user requested to move `favicon.ico` correctly, ensure it's copied to `dist` on build, and revise the backend to handle the file correctly.

**Original Branch Commit ID:**
f0c9637875640fd7ee0294f3e87ea0cf51212728

**Root Cause:**
1. The `favicon.ico` file was placed in the project root directory rather than `public/`, so Vite's build process (`npm run build`) was unaware of it and did not automatically copy it to `dist/`.
2. The `index.html` lacked the meta tag explicitly defining the favicon location.
3. The FastAPI backend lacked a specific route to handle `GET /favicon.ico` before the fallback catch-all route for the Single Page Application (`index.html`).

**Solution:**
1. Moved `favicon.ico` to the `public/` folder so `vite build` correctly outputs it to the `dist/` root directory.
2. Appended `<link rel="icon" type="image/x-icon" href="/favicon.ico" />` to `index.html`.
3. Created a dedicated `@app.get("/favicon.ico")` route in `backend/main.py` using `FileResponse` to handle fetching the image from `dist/favicon.ico`.
