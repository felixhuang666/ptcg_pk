# Change Summary - TID: 514

## Prompt Summary
分析目前的Code base，將怪獸對戰遊戲改為雙人連線對戰的回合制，並加強美工與戰鬥平衡。

## Original Branch Commit ID
N/A (Initial changes in current session)

## Root Cause
- 遊戲原本使用 ATB (Active Time Battle) 即時制，較不具備回合制的策略感。
- 戰鬥傷害公式為簡單的 `ATK - DEF`，容易造成傷害過低 (1點) 或數值膨脹問題。
- 怪獸血量偏低，導致戰鬥結束過快。
- UI 使用表情符號 (Emoji) 作為佔位符，視覺效果較為陽春，且缺乏戰鬥動畫反饋。

## Solution
1. **重構遊戲引擎為回合制**：
   - 將即時 AP 增加機制改為回合階段制（擲骰、行動、結算）。
   - 根據速度 (SPD) 決定行動順序。
   - 玩家在自己的回合內選擇技能或重新擲骰。
2. **戰鬥平衡優化**：
   - 怪獸生命值 (HP) 全面提升 5 倍，延長對戰策略空間。
   - 引入百分比減傷公式：`Damage = ATK * (100 / (100 + DEF))`，確保傷害平滑縮放。
   - 重新調整怪獸基礎數值 (STR, CON, DEX)。
3. **美工與視覺強化**：
   - 導入 `framer-motion` 實現戰鬥動畫（怪獸登場、衝擊攻擊、受傷抖動）。
   - 將 Emoji 替換為自定義的 SVG 怪獸圖標，並為不同屬性區分顏色。
   - 實作「飄浮傷害數字」特效，提升打擊感。
   - 優化戰鬥介面，包含明確的回合指示器與血條動畫。
4. **連線對戰優化**：
   - 確保 Socket.IO 狀態同步正確處理回合切換。
   - 支援自動戰鬥模式在回合制下的運作。

## Changed Files
- `src/shared/types.ts`: 增加 `GamePhase` 型別與 `GameState` 回合相關欄位。
- `server.ts`: 移除計時器，實作回合制狀態機。
- `src/shared/gameLogic.ts`: 更新傷害公式與技能結算邏輯。
- `src/shared/gameData.ts`: 調整全怪獸數值平衡。
- `src/components/Battle.tsx`: 重大 UI 更新，加入動畫與 SVG 視覺。
- `docs/game_design.md`: 更新遊戲設計文件。
