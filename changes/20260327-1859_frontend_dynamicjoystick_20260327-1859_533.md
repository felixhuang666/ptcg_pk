**Root Cause:**
The original virtual gamepad was statically positioned at (80, 400). Its interactions were hardcoded to trigger only when clicking within a fixed distance (120 pixels) from this static center. If the user dragged their pointer outside of the browser canvas while controlling the joystick, the `pointerout` event would immediately reset the gamepad state, freezing the character controls until clicked again. Additionally, the cross graphic for the D-pad was drawn on a static graphics object, making it immovable.

**Solution:**
Modified `src/components/RpgMode.tsx` to implement a dynamic joystick:
- Configured the joystick base, thumb, and cross graphics to be hidden by default (`setVisible(false)`).
- Hooked up `pointerdown` listener on the Phaser scene to move these UI elements directly to the pointer's coordinates `(pointer.x, pointer.y)` when triggered, making them visible and activating movement controls.
- Prevented UI objects from triggering the joystick by checking `gameObjects.length`.
- Re-drawing the joystick graphics cross dynamically based on the pointer down coordinates.
- Reset the joystick controls on `pointerup` and `gameout` (rather than standard `pointerout` to ensure continuous tracking even if pointer leaves interactive objects within the game, stopping only when leaving the actual game view or mouse release).
