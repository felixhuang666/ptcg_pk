## Root Cause
The project lacked a structured testing environment for both frontend and backend, missing required dependencies, configurations, test folders, and a clear method to execute them via the Makefile. Additionally, testing authenticated backend routes was difficult without a dedicated test user.

## Solution
1. **Frontend Testing:** Installed `vitest`, `@testing-library/react`, and `jsdom`. Created `vitest.config.ts`, `tests/setup.ts`, and added a basic UI rendering test in `tests/App.test.tsx`.
2. **Backend Testing:** Utilized existing `pytest` setup. Created `backend/tests/test_main.py` using `httpx` to verify backend API auth routes (`/api/auth/me`, `/api/auth/logout`).
3. **Tester User:** Added an `/auth/dev_login` endpoint to `backend/main.py` to bypass real Google OAuth, generate fake OAuth session data for a `tester` user, and set the `session_id` cookie locally.
4. **Build Automation:** Updated `Makefile` with `test`, `test-frontend`, and `test-backend` rules. Modified `package.json` to include `"test": "vitest run"`.
