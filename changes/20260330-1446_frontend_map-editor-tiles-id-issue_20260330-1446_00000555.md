## Root Cause
1. In `src/components/RpgMapEditor.tsx`, the editor's left sidebar and right sidebar tile detail views assumed that the metadata `id` found in `/assets/map_tileset/*.json` files was 0-indexed (e.g. `t.id === id`), but the sample JSON file `dist/assets/map_tileset/main_20x10.json` indicated that `id` begins at 1.
2. The drawing logic passed the 1-based tile ID to Phaser, but Phaser stored it as the array index which should be 0-based. This caused selecting tile ID 1 to render the block for tile ID 2.
3. The application lacked a dedicated UI refresh button to reload the available tileset JSON files dynamically.
4. The "進入全螢幕模式" overlay was hardcoded and intrusive for local desktop/web development where users might not want fullscreen.
5. The map editor had no dedicated "Move/Pan" mode outside of using middle/right click.
6. The toolbar was cluttered with long text strings for standard actions like save, undo, and generate.

## Solution
1. **Fix Tile ID Issue**:
    - Updated `RpgMapEditor.tsx` to align the loop index (`id`, which is 0-indexed) with the 1-indexed metadata ID. Modified the lookup: `t.id === id + 1`.
    - Modified the fallback object in `tileMeta` to also assign `id: id + 1` so it matches the 1-indexed expectation.
    - Adjusted the drawing math in `MainScene.updateMapData`. Since `this.currentTileType` is passed from the UI as 1-indexed (`scene.currentTileType = id + 1`), the internal array assignment subtracts 1: `targetVal = this.isEraser ? -1 : this.currentTileType - 1;`
2. **Add Refresh Icon**:
    - Extracted the tileset fetch call into a reusable function `loadTilesets`.
    - Integrated a `RefreshCw` icon button in the left sidebar next to the "Tilesets" title, configured with an `onClick` handler pointing to `loadTilesets()`.
3. **Remove Fullscreen Prompt**:
    - Removed the `showFullscreenPrompt` state and the associated intrusive React overlay JSX entirely.
4. **Draw/Move Mode Toggle**:
    - Added an `editorMode` state (`'draw' | 'move'`) and a toggle button with `Hand` and `Pencil` Lucide icons.
    - Modified the `MainScene` pointer interactions so when `editorMode === 'move'`, dragging the pointer behaves like middle/right click panning (`scene.cameras.main.scrollX -= (pointer.x - pointer.prevPosition.x)`).
5. **Icon Button Toolbar Cleanup**:
    - Swapped out "Undo", "Redo", "New Empty Map", "Save Map", and "Generate" text buttons for Lucide icons (`Undo2`, `Redo2`, `FilePlus`, `Save`, `Sparkles`). Added hover tooltips (`group-hover:opacity-100`) for discoverability.
6. **Verification**:
    - Passed E2E Playwright tests verifying the modes, toolbar layout changes, and visual styling of the editor.
