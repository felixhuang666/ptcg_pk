## Root Cause
1. In `src/components/RpgMapEditor.tsx`, the editor's left sidebar and right sidebar tile detail views assumed that the metadata `id` found in `/assets/map_tileset/*.json` files was 0-indexed (e.g. `t.id === id`), but the sample JSON file `dist/assets/map_tileset/main_20x10.json` indicated that `id` begins at 1.
2. Because the metadata was accessed via `t.id === id`, the tiles were offset by 1, returning undefined or incorrect `tileMeta` mapping when rendering the background position calculation.
3. The application lacked a dedicated UI refresh button to reload the available tileset JSON files dynamically.

## Solution
1. **Fix Tile ID Issue**:
    - Updated `RpgMapEditor.tsx` to align the loop index (`id`, which is 0-indexed) with the 1-indexed metadata ID. Modified the lookup: `t.id === id + 1`.
    - Modified the fallback object in `tileMeta` to also assign `id: id + 1` so it matches the 1-indexed expectation.
    - Adjusted the background offset math in the right sidebar. Since `selectedTileData.id` is now always 1-indexed, subtracted 1 (`(selectedTileData.id - 1)`) to determine the correct 0-indexed grid coordinates.
2. **Add Refresh Icon**:
    - Extracted the tileset fetch call (`fetch('/api/map/tilesets')`) into a reusable function `loadTilesets`.
    - Integrated a `RefreshCw` icon button in the left sidebar next to the "Tilesets" title, configured with an `onClick` handler pointing to `loadTilesets()`.
3. **Verification**: Successfully used a Playwright E2E visual test script to ensure the Map Editor renders with the refresh icon and accurately calculates the background positions for both the tile grid and the details view.
