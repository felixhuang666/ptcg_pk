# Fix Monster Rendering in RPG Mode

## User Prompt
在RPG模式下, monstor cannot be rendered

i try to switch to local_scene_1 (which is from local asset), i can see the map be rendered, but the game objects cannot be rendered correctly.
- check boss_single_eye_orc in local_scene_1

## Original Commit ID
N/A

## Root Cause
In `src/components/RpgMode.tsx`, the game object loading and instantiation logic heavily relied on the `sprite_sheets` property from the game object template. For monsters like `boss_single_eye_orc`, their template lacks `sprite_sheets` entirely and instead uses `default_image`. The existing code did not attempt to load `default_image` or instantiate it into a Phaser sprite if `sprite_sheets` was missing, resulting in missing textures/sprites in the RPG scene.

## Solution
Modified `src/components/RpgMode.tsx`:
1. Updated the preload loop to check for and load `default_image` into a `game_obj_img_{key}` texture if `sprite_sheets` is empty.
2. Updated the instantiation loop to use the `game_obj_img_{key}` texture key if no `sheetMeta` is available.
3. Added logic to set the correct display sizes based on the `container_override` property on the instance or `container_width`/`container_height` on the template when rendering a static default image.
