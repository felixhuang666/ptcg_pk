# Map Editor Enhancements & Procedural Generation

- **Prompt**: "新增 動態產生 按鈕 可以觸發重新生成地圖. 在 FastAPI 中實作 Perlin Noise，可新增一張世界地圖, 也要填充適合的素材 (參考public/assets/map_tileset/cute_tileset.json). 在地圖管理介面, 可以選擇地圖, 可以重新命名地圖. 新增切換地圖的功能 於上方工具列中. 使用者放大或是縮小, 不能影響工具列大小, 工具列寬度永遠等於視窗寬度, 當寬度不足, 自動縮小元件大小來配合視窗寬度. 參考public/assets/map_tileset/cute_tileset.json, 這裡描述可用的地圖素材, 圖片資料參考 public/assets/map_tileset/cute_tileset.png"
- **Original Branch Commit ID**: 5249fc54287db994ed948bcacade7168a420848f
- **Root Cause**: The RPG mode editor previously lacked map management (create/rename/switch) and had a hardcoded 3-color basic tileset generated programmatically. There was no procedural map generation available.
- **Solution**:
  1. Updated the backend `backend/main.py` to add `in_memory_maps` and added endpoints for map management (`GET /api/maps`, `GET /api/map/{id}`, `POST /api/map/{id}`).
  2. Implemented `POST /api/map/generate` utilizing the `noise` library's `pnoise2` function to generate a 200x200 map seeded procedurally. The heightmap values are mapped to tiles `2` (Grass), `48` (Water), and `94` (Mountain) using `cute_tileset.json`.
  3. Created a DB Migration script `scripts/migrate_20260327_1754_add_map_name.py` for adding a `name` column to the Supabase `maps` table.
  4. Updated `RpgMode.tsx` to preload and utilize `cute_tileset.png`.
  5. Implemented a fixed, absolute-positioned Top Toolbar in React for map selection, map renaming, and generating a new procedural map. The toolbar is responsive and adapts to the window width, remaining completely unaffected by Phaser's camera zoom/pan.
