# Task 575: Fix Uncaught TypeError when switching RPG play mode and map editor

**Prompt:**
Objective:when switch RPG play mode and map editor, see error sometimes

index-M461uwCg.js:1571 Uncaught TypeError: Cannot read properties of null (reading 'add')
    at initialize.graphics (index-M461uwCg.js:1571:88)
    at l.toggleGrid (index-M461uwCg.js:6521:2116)
...

**Original Branch Commit ID:**
fc21650f716dd270f9d0ff3038766ce9e910104e

**Root Cause:**
When a user switches between the RPG map mode (`RpgMode.tsx`) and the RPG map editor (`RpgMapEditor.tsx`), the underlying Phaser scene is destroyed upon the component unmounting. However, the global reference to the scene `(window as any).__PHASER_MAIN_SCENE__` is not cleared.
When the new component mounts, its `useEffect` hook triggers a grid setting update which calls `scene.toggleGrid` on the globally stored reference. Because that reference points to the old, destroyed scene, accessing `this.add` (and consequently `this.add.graphics()`) inside `toggleGrid` throws an `Uncaught TypeError: Cannot read properties of null (reading 'add')`.

**Solution:**
1. Added a check inside `toggleGrid` in `src/components/RpgMapEditor.tsx` to exit early if the scene has been destroyed: `if (!this.sys || !this.sys.game || !this.add) return;`
2. Updated the component unmount cleanup functions in both `src/components/RpgMode.tsx` and `src/components/RpgMapEditor.tsx` to explicitly clear the global reference if it points to the current scene:
```typescript
if ((window as any).__PHASER_MAIN_SCENE__ === mainSceneRef.current) {
  (window as any).__PHASER_MAIN_SCENE__ = null;
}
```
This ensures that any newly mounted components do not attempt to call methods on a stale, destroyed Phaser scene instance.
