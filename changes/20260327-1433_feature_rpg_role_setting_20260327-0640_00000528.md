## Root Cause / Requirements Analysis
The user wants to allow players to select their character avatar from a list of predefined roles in a new `game_roles` table, and have this selection reflected in the RPG Mode sprites.

## Solution
1. **Database Migration Script**: Created `scripts/migrate_20260327-1433_create_game_roles.py` to create the `game_roles` table and insert the default "魔女" role data.
2. **Backend API**: Added a new REST endpoint `GET /api/roles` in `backend/main.py` that fetches from the Supabase table, with fallback to the default role if the table is unavailable.
3. **Store Update**: Updated Zustand store `src/store/appStore.ts` to manage `roles` and `selectedRoleId`.
4. **App Level Fetch**: Added a `fetch('/api/roles')` call in `src/App.tsx`'s initial `useEffect` to populate the global store before entering RPG Mode.
5. **Role Settings Component**: Implemented `src/components/RoleSetting.tsx` allowing users to view and select roles, with visual representation of the walking and attacking sprites. Added a button on the main menu to access it.
6. **RPG Mode Integration**: Updated `src/components/RpgMode.tsx` to read `selectedRoleId` from the store and pass the correct sprites into the Phaser scene's `preload()` step.
