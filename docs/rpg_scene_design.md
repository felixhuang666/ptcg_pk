# RPG Scene Editor Design

## 1. Overview
The RPG Scene Editor is a unified tool for creating and editing comprehensive RPG Scenes within "Monster Battle". An RPG Scene serves as a macro-level container that orchestrates multiple maps, NPCs, items, game objects, and events to create a cohesive game world for players to explore.

The editor combines traditional map painting capabilities with scene composition, allowing designers to stitch individual maps together, populate them with interactive entities, and define complex game logic.

## 2. RPG Scene Structure
A RPG Scene is not a single map; rather, it is a unified coordinate space where multiple maps and entities are placed.

### Table Schema
```sql
create table public.game_scene (
  id integer generated by default as identity not null,
  created_at timestamp with time zone not null default now(),
  name text not null,
  map_list jsonb null,
  scene_entities jsonb null,
  constraint game_scene_pkey primary key (id)
) TABLESPACE pg_default;
```

### 2.1 map_list
The `map_list` is a JSONB array that defines the spatial arrangement of individual maps within the global scene coordinate system. Each map acts like a "chunk" of the world. Maps are associated with specific layers via `layer_id` and are dynamically sorted during rendering based on the index order defined in the `sceneData.layers` array. Note: For backwards compatibility, the code may also check `scene_entities.map_list`.

```json
{
  "map_list": [
    {
      "map_id": "map_1",
      "map_size": { "width": 40, "height": 40 },
      "offset_position": { "x": 0, "y": 0 }
    },
    {
      "map_id": "map_2",
      "map_size": { "width": 20, "height": 40 },
      "offset_position": { "x": 40, "y": 0 }
    }
  ]
}
```

### 2.2 scene_entities
The `scene_entities` JSONB structure holds instances of all dynamic objects, items, NPCs, and events placed globally within the scene coordinates.

```json
{
  "layers": [...],
  "npcs": [
    { "id": "npc_1", "type": "merchant", "position": { "x": 15, "y": 20 } }
  ],
  "items": [
    { "id": "item_1", "type": "health_potion", "position": { "x": 5, "y": 5 } }
  ],
  "events": [
    { "id": "evt_1", "trigger_type": "on_step", "action": "teleport", "position": { "x": 39, "y": 20 } }
  ]
}
```

## 4. Layer Integration
The Scene Editor respects the 6-layer architecture of individual maps (`base`, `decorations`, `obstacles`, `objectCollides`, `objectEvent`, `topLayer`), but introduces a new paradigm for dynamic entities.

### Integration Approach
* **Static Map Layers (Layers 1-6)**: Loaded from individual map definitions based on the `map_list`. Rendered statically via Phaser tilemaps.
* **Dynamic Scene Layer**: A virtual 7th layer managed by the Scene Editor. This layer contains Phaser Sprites and Objects (NPCs, Items) that have their own update loops and physics bodies.
* **Depth Sorting (Z-Index)**: Dynamic scene objects are depth-sorted based on their global Y-coordinate to ensure they render correctly in front of or behind map features (like trees on the `objectCollides` layer). The map's `topLayer` always renders above everything else to maintain dynamic transparency.

## 5. UI Design (Current Implementation)

- **Top Bar (Global Controls)**:
  * New Scene, Save Scene
  * Export/Import
  * Panel toggles (Left/Right sidebars)
  * Fullscreen toggle

- **Left Sidebar**:
  * Scene list (dropdown)
  * Map palette for adding maps to scene
  * Game Object templates

- **Center (Canvas)**:
  * Phaser canvas with infinite pan/zoom
  * Grid overlay
  * Drag-and-drop map placement
  * Layer z-ordering

- **Right Sidebar (Property Inspector)**:
  * Context-sensitive panel
  * Map properties: offset position
  * Game Object properties: position, properties
