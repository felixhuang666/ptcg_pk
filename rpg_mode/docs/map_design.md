# Map Design Document (地圖設計文件)

## 1. 系統概述 (Overview)
本遊戲採用基於網格 (Grid-based) 的 2D 區塊地圖系統。地圖資料在伺服器端與所有連線的玩家之間保持即時同步，當有玩家在編輯模式下修改地圖並儲存時，所有玩家的畫面都會即時更新。

## 2. 地圖資料結構 (Data Structure)
地圖資料以 JSON 格式儲存與傳輸，其基本結構如下：

```json
{
  "width": 20,
  "height": 15,
  "tiles": [0, 0, 1, 2, ...] 
}
```
*   **`width`**: 地圖的橫向區塊數量（預設為 20）。
*   **`height`**: 地圖的縱向區塊數量（預設為 15）。
*   **`tiles`**: 一維陣列，用來表示 2D 的地圖網格（由左至右、由上至下排列）。陣列長度為 `width * height`，陣列中的每個整數代表一個特定的地形種類。

## 3. 地形種類 (Tile Types)
目前遊戲支援三種基本地形，在畫面上以不同顏色的矩形呈現 (每個區塊大小為 32x32 像素)：

| ID | 名稱 (Name) | 顏色 (Color) | 說明 (Description) |
|:---:|:---|:---|:---|
| **0** | 草地 (Grass) | `#228B22` (綠色) | 基本的平地地形。 |
| **1** | 水域 (Water) | `#1E90FF` (藍色) | 水面地形（目前可通行，未來預計加入碰撞阻擋）。 |
| **2** | 山脈 (Mountain) | `#808080` (灰色) | 高山地形（目前可通行，未來預計加入碰撞阻擋）。 |

## 4. 遊戲模式與地圖編輯器 (Game Modes & Map Editor)
遊戲分為兩種主要模式，玩家可透過畫面**右下角**的切換按鈕 (Mode Toggle Button) 進行切換：

### 4.1 遊玩模式 (Play Mode)
*   預設模式，玩家可以在世界地圖中移動與其他玩家互動。
*   支援滑鼠滾輪縮放 (Zoom-in / Zoom-out)，鏡頭會自動跟隨玩家。
*   右上角會顯示當前地圖名稱與玩家的網格座標。

### 4.2 編輯模式 (GM Editor)
允許擁有權限的玩家（目前為開放測試）即時繪製地圖。
*   **操作方式**：
    *   按下鍵盤 `1`：選擇「草地」筆刷。
    *   按下鍵盤 `2`：選擇「水域」筆刷。
    *   按下鍵盤 `3`：選擇「山脈」筆刷。
    *   **滑鼠左鍵點擊或拖曳**：將選擇的地形繪製到地圖上。
    *   **滑鼠右鍵/中鍵拖曳** 或 **方向鍵**：平移地圖鏡頭。
    *   **滑鼠滾輪**：縮放地圖 (Zoom-in / Zoom-out)。
*   **編輯功能**：
    *   支援復原 (Undo) 與重做 (Redo) 功能。
    *   右上角會顯示當前地圖名稱與鏡頭中心的網格座標。
*   **儲存機制**：
    *   點擊畫面下方的 `[ SAVE MAP ]` 按鈕後，會透過 `POST /api/map` 將最新的地圖資料發送給伺服器。
    *   伺服器接收後，會透過 Socket.io 廣播 `map_updated` 事件，讓所有在線玩家的地圖瞬間同步。

## 5. 未來擴充計畫 (Future Enhancements)
1.  **碰撞系統 (Collision System)**：為「水域」與「山脈」加入物理碰撞邊界，讓角色無法穿透這些地形。
2.  **圖塊素材 (Tilesets)**：將目前的純色矩形替換為真實的圖片素材 (Sprite-based tiles)，提升視覺表現。
3.  **多地圖支援 (Multiple Maps)**：實作地圖切換功能，允許玩家走到地圖邊緣時傳送到另一張地圖（例如：村莊進入森林）。
4.  **動態地圖大小 (Dynamic Sizing)**：在編輯器中加入調整地圖寬高 (Width/Height) 的功能。
5.  **物件層 (Object Layer)**：除了地形之外，新增一層用來放置樹木、房屋、NPC 或道具的獨立圖層。
